# Meetly

## Cursor Cloud specific instructions

### Overview

Meetly is a 1:1 video calling app on Cloudflare Workers with real-time speech translation. Single package (not a monorepo) using npm. See `README.md` for full architecture and scripts reference.

### Running locally

- **`npm run dev`** starts everything: builds the client via `predev` hook, then runs `wrangler dev` on **port 8787**.
- Wrangler provides the Worker, Durable Objects (local), Workers AI (remote proxy), and static asset serving in a single process. No Docker or external databases needed.
- `.dev.vars` must contain `ELEVENLABS_KEY`. Copy from `.dev.vars.example` if missing. The key is also available as an environment variable `ELEVENLABS_KEY`.

### Type checking / lint

- **`npx tsc --noEmit`** — TypeScript check (run `npx wrangler types` first if `worker-configuration.d.ts` is stale).
- No ESLint or other linter is configured in this project.

### Building

- **`npm run build:client`** — bundles `client/app.jsx` → `public/app.js` via esbuild.
- **`npm run build`** — alias for `build:client`.

### Key caveats

- Workers AI binding (`env.AI`) always connects to Cloudflare's remote inference, even in local dev. This requires internet access and a Cloudflare account.
- The client bundle (`public/app.js`) is committed but also regenerated by `predev`. After editing `client/app.jsx`, the dev server auto-rebuilds on restart, or run `npm run build:client` manually.
- WebRTC features (camera/mic) require a real browser with media device access; headless testing will show "Camera/mic permission required" but the app still functions for API and WebSocket testing.
